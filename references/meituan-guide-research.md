# 美团攻略（guide.meituan.com）调研方法

> **v1.2.0 新增**。美团官方 2026 年 4 月上线的"编辑版"攻略（`guide.meituan.com`），**数据来源就是大众点评公开数据 + 媒体评测 + 真实口碑**。**反爬对抗 = 0**（编辑页是静态的，AI 用 `WebFetch` 直接拿）。
>
> **定位**：餐厅候选池的**第一轮筛选器**。**不替代高德 POI 详情和小红书软信号**——三者是组合关系。
>
> **何时用**：用户做国内 8 大热门城市（上海/北京/成都/广州/深圳/杭州/重庆/武汉）的旅行时，**首选**用美团攻略定"该城市该品类有什么好店"。

---

## 1. URL 模式（AI 必须自己观察，不硬编码）

### 1.1 已知稳定的 URL 模式

| 内容类型 | URL 模式 | 例子 |
|---------|---------|------|
| 城市美食攻略 | `https://guide.meituan.com/<city>/canyin` | `https://guide.meituan.com/chengdu/canyin` |
| 城市住宿攻略 | `https://guide.meituan.com/stay/<city>` | `https://guide.meituan.com/stay/wuhan` |
| 通用省钱攻略 | `https://guide.meituan.com/save-money` | — |
| 城市拼音 | 全小写 | shanghai / chengdu / beijing / guangzhou / shenzhen / hangzhou / chongqing / wuhan |

### 1.2 未知时的探查姿势

> **不要硬编码 URL 模板**——美团可能会改路径。AI 拿到后用以下姿势探：

```
1. 先试 guide.meituan.com/<拼音>/canyin
2. 拿不到 → 试 guide.meituan.com/<拼音>（让美团 404 / 重定向告诉你结构）
3. 拿到后从页面 HTML 里找 "美食"/"餐厅"/"火锅" 等关键词锚点
4. WebFetch 失败/超时 → 改用高德 text_search 兜底（详见 references/dianping-research.md §1）
```

> **遇到 WebFetch 超时**：**不要反复重试**（美团服务器可能限速）。**改用** `mcp__playwright__browser_navigate` + `browser_snapshot` 走真实浏览器拿（这是合法用法——只是拿一个公开页）。**或**直接降级到"高德 text_search 按菜系搜"。

## 2. 页面内容结构（典型）

拿到一个城市的"怎么吃"页（`guide.meituan.com/<city>/canyin`），通常包含：

### 2.1 头部
- 城市概览（消费水平、餐饮门店数、点评量）
- 季节性提醒（"五一/国庆需提前预订"）

### 2.2 品类分块
- **火锅**、**烤肉**、**日料**、**西餐**、**甜品**、**粤菜**、**川菜**…
- 每个品类下 5-20 家**编辑推荐店**
- 推荐店通常带：**店名** + **区域** + **价位** + **入选理由标签**

### 2.3 场景分块
- **商务**、**约会**、**家庭聚餐**、**朋友聚会**…
- 跨品类的场景化推荐

### 2.4 价格梯度
- 50 元以下 / 50-100 / 100-200 / 200+ 各档推荐

### 2.5 关键词标签
- "本地人爱去" / "周末排队" / "工作日推荐" / "不踩雷" / "氛围感" / "必吃榜水平"…

### 2.6 城市覆盖
- **8 大城市**有详细攻略（火锅/烤肉/日料等多品类）：上海、北京、成都、广州、深圳、杭州、重庆、武汉
- **50+ 城市**有概要（消费水平 + 总体建议）
- 其他城市**不一定有"怎么吃"详细页**——遇到 404 就改用高德 text_search

## 3. 怎么用

### 3.1 AI 调用姿势

```python
# 1. WebFetch 拿编辑页
WebFetch: https://guide.meituan.com/chengdu/canyin

# 2. AI 解析页面 → 提取:
#    - 火锅品类下 5-20 家编辑推荐店（店名 + 区域 + 价位 + 标签）
#    - 场景分类（约会/家庭/商务）
#    - 关键词标签（"本地人爱去" / "周末排队"等）

# 3. 对清单里每家店：
mcp__amap__maps_text_search(keywords="<店名>", city="成都")
# 4. 拿 POI ID → maps_search_detail 拿坐标/营业/评分

# 5. V3 通勤 + 时间匹配 + 标签筛选 → 主推 vs 备选
```

### 3.2 输出模板

```md
## 餐厅候选池（来源：美团攻略 + 高德 POI 详情）

**主推**：蜀大侠火锅（科华北路店）
- 来源：美团攻略"火锅"品类 + "本地人爱去"标签
- 高德 POI：评分 4.6，人均 120，距主区域 0.1 km，营业至 22:00

**备选 1**：小龙坎老火锅（春熙路店）
- 来源：美团攻略"火锅"品类 + "周末排队"标签
- 高德 POI：评分 4.5，人均 110，距主区域 0.4 km，营业至 23:00

**备选 2**：川西坝子火锅（春熙路总店）
- 来源：美团攻略"火锅"品类 + "不踩雷"标签
- 高德 POI：评分 4.4，人均 90，距主区域 0.3 km，营业至 22:30
- **小红书避雷**：近期 2 篇"排队太久"笔记
```

### 3.3 标签的解读

| 标签 | AI 解读 |
|------|--------|
| "本地人爱去" | 强推荐（高优先级入选） |
| "周末排队" | 时间提示：建议错峰/工作日午市 |
| "工作日推荐" | 反向提示：周末**不**去 |
| "不踩雷" | 强推荐（高优先级入选） |
| "氛围感" | 软信号维度（约会/拍照） |
| "必吃榜水平" | 评分高 + 编辑筛选（不一定上榜） |
| "游客店" | **反向避雷**（用户说"不要游客店"时直接剔）|

## 4. 局限与应对

| 局限 | 应对 |
|------|------|
| 8 大城市之外内容稀疏 | 改用高德 text_search 按菜系搜做兜底 |
| 编辑筛选≠用户偏好 | **只做候选池筛选器**，不替代用户最终决定 |
| 没有"实时排队" | 小红书搜"<店名> 排队"+ 标注"工作日午市/周末晚市排队风险" |
| 拿不到必吃榜入选证明 | 主轨 1 拿评分（高德评分 4.6+ 通常对应必吃榜水平）|
| URL 模式可能变 | **AI 拿到后自己观察**：URL 改了就用新模式（不要硬编码 URL 模板）|
| WebFetch 偶尔超时 | 不要反复重试，改用 Playwright 走真实浏览器（合法用法），或直接降级到高德 text_search |

## 5. 与其他数据源的组合

| 场景 | 美团攻略 | 高德 POI 详情 | 小红书软信号 |
|------|---------|------------|------------|
| 城市/品类候选池 | ✅（**主用**）| 兜底 | ❌ |
| 坐标/营业时间/类型 | ❌ | ✅（**主用**）| ❌ |
| 评分（粗筛）| 编辑标签 = 入选 | ✅ | ❌ |
| 排队实况 | 部分 | ❌ | ✅（**主用**）|
| 避雷/踩雷 | ❌ | ❌ | ✅（**主用**）|
| 氛围/拍照 | 部分 | ❌ | ✅（**主用**）|
| 当地人偏好 | ✅（"本地人爱去"标签）| ❌ | ✅ |

**典型组合（成都 3 天 Day 2 午餐）**：

```
1. WebFetch guide.meituan.com/chengdu/canyin
   → 火锅品类下 5-10 家编辑推荐
2. 对每家店 → mcp__amap__maps_text_search + maps_search_detail
   → 拿坐标/营业/评分 → 算 V3
3. 对剩 3-5 候选 → xhs-skill 搜 "<店名> 排队"/"<店名> 避雷"
4. 出主推 + 备选
```

## 6. 完整示例

```python
# 假设：用户做成都 3 天，Day 2 主区域春熙路，午餐想吃火锅

# Step 1：美团攻略定方向
WebFetch: https://guide.meituan.com/chengdu/canyin
# → 拿到火锅品类 8 家编辑推荐：蜀大侠、大龙燚、皇城老妈、川西坝子、小龙坎、电台巷、谭鸭血、牛华绿叶

# Step 2：高德 POI 详情填硬信号
# 蜀大侠火锅（科华北路店）
mcp__amap__maps_text_search(keywords="蜀大侠火锅 成都", city="成都")
mcp__amap__maps_search_detail(id="B0JA5UF6R*")
# → {"rating": "4.6", "cost": "120", "opentime2": "11:00-14:00,17:00-22:00", "location": "104.081,30.659"}

# ... 对其他 7 家重复

# Step 3：V3 通勤匹配（春熙路酒店 104.081,30.658）
# 蜀大侠 → 0.1km ✅
# 大龙燚（科华北路店）→ 0.5km ✅
# 皇城老妈（总府路店）→ 0.3km ✅
# 川西坝子（春熙路总店）→ 0.4km ✅
# 小龙坎（春熙路店）→ 0.2km ✅
# 电台巷火锅（春熙路店）→ 0.6km ✅
# 谭鸭血（春熙路店）→ 0.3km ✅
# 牛华绿叶（春熙路店）→ 0.7km ✅ （8 家全过 V3）

# Step 4：标签筛选（要"本地人爱去" + "不踩雷"）
# 蜀大侠 → "本地人爱去" ✅
# 大龙燚 → "周末排队" 标签
# 皇城老妈 → "必吃榜水平" 标签
# 川西坝子 → "不踩雷" ✅
# 小龙坎 → 无特殊标签
# 电台巷 → "工作日推荐"（周末不去）
# 谭鸭血 → "氛围感"
# 牛华绿叶 → "游客店" ⚠️（用户说不要游客店 → 剔）

# Step 5：营业时间 + 评分
# 蜀大侠：4.6 分 + 营业至 22:00 ✅
# 川西坝子：4.4 分 + 营业至 22:30 ✅
# 大龙燚：4.5 分 + 营业至 23:00 ✅

# Step 6：小红书软信号
# 蜀大侠：周末晚市排队 30min+；工作日午市无需排队 ✅
# 大龙燚：近期 1 篇"服务下降" ⚠️
# 川西坝子：近期多人推"毛肚" + 鸳鸯锅 ✅

# Step 7：主推 vs 备选
# 主推：蜀大侠（评分高 + 本地人爱去 + 距主区域极近 + 标签 + 软信号都好）
# 备选：川西坝子（不踩雷 + 营业晚 + 距离近）
# 不推：牛华绿叶（游客店标签）
```

## 7. 何时不用美团攻略

- **8 大城市之外的国内城市**：直接用高德 `text_search` 按菜系搜（候选池可能 100+，**多花 10 分钟做 V3 过滤**）
- **境外旅行**：完全用不了（美团攻略无境外内容），改用：
  - **日本**：Tabelog（WebFetch 抓 M 站）
  - **东南亚/欧美**：Google Maps Places API（通过 Maps MCP 拿评分/评价数）
  - **港澳台**：OpenRice（WebFetch）或高德
- **用户明确说"就要必吃榜上榜"**：跳过美团攻略，**直接装 opencli 深度档**（参见 `references/dianping-research.md` §6）
